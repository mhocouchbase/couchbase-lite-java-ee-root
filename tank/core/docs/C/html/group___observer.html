C4DatabaseObserver* c4dbobs_create ( C4Database *  database,
C4DatabaseObserverCallback  callback,
void *  context 
)

Creates a new database observer, with a callback that will be invoked after the database changes.

The callback will be called once, after the first change. After that it won't be called again until all of the changes have been read by calling c4dbobs_getChanges.

Parameters
databaseThe database to observer.
callbackThe function to call after the database changes.
contextAn arbitrary value that will be passed to the callback.
Returns
The new observer reference.

§ c4dbobs_getChanges()

uint32_t c4dbobs_getChanges ( C4DatabaseObserver *  observer,
C4DatabaseChange  outChanges[],
uint32_t  maxChanges,
bool *  outExternal 
)

Identifies which documents have changed since the last time this function was called, or since the observer was created.

This function effectively "reads" changes from a stream, in whatever quantity the caller desires. Once all of the changes have been read, the observer is reset and ready to notify again.

Parameters
observerThe observer.
outChangesA caller-provided buffer of structs into which changes will be written.
maxChangesThe maximum number of changes to return, i.e. the size of the caller's outChanges buffer.
outExternalWill be set to true if the changes were made by a different C4Database.
Returns
The number of changes written to outDocIDs. If this is less than maxChanges, the end has been reached and the observer is reset.
